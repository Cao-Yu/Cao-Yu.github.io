Move Zeroes

原题地址：Move Zeroes

问题描述：移除数组中的零，并且保持元素的顺序，且不能建立新的数组。

My Solution

遍历数组，当元素为零时，查找下一个非零元素，交换元素并且更新指针。

Better Solution

遍历数组，如果元素不为零，与 p 所指元素交换。

此法用索引 p 来标记数组的有效元素，如果有元素不为零，则赋值给有效位置。

总结

我的思路还算可以接受，但是代码过于冗杂。而且内嵌了 while 循环，增加了计算量。Better Solution 中的方法虽然每次遍历非零元素都要与自己交换一次，但是省去了判断 0 的个数和内层循环，尤其是省去了判断末尾还有没有零这个操作。

Rotate Array

原题地址：Rotate Array

问题描述：给定数组和一个索引，将数组分成两份，交换位置。

My Solution

不提也罢

Better Solution

先定义一个空间复杂度为 O(1) 的 swap 函数，用于水平翻转给定范围的数组。

第一步先将整个数组水平翻转，然后再分别翻转前段儿和后端儿。

总结

看似复杂的问题，背后的规律可能很简单。遇到问题首先视图分析出问题背后的规律。

Remove Element

原题地址：Rotate Array

问题描述：给定一个数组和一个值，将数组中所有与该值相等的元素删去并且返回新数组的长度，可以乱序。

My Solution

不提也罢

Better Solution

定义两个指针，从头开始遍历数组。如果与给定元素想等，则交换并且右指针左移。如果元素不等，左指针右移。

总结

首先，在不明确执行次数的情况下，while 循环比 for 循环更简单。

其次，要善于利用已知条件，题中明确可以乱序，所以首先考虑交换元素。
