iOS开发-UITextView

Veröffentlicht am 2016-04-24   |   in iOS开发

UITextView 是 iOS 中的文本输入、显示控件。与 UITextField 不同的是，UITextView 是多行的文本域，支持滚动，多用于显示长篇内容。而 UITextField 是文本框，多用于输入用户名、密码等。

值得一提的是，UITextView 是 UIScrollView 的子类，具有滚动视图的行为和属性。

如果要在 UITextView 中输入文字，需要先设置 editable 为 true ，然后再将 textView 设置为第一响应者。

使用

let textView = UITextView(frame:view.frame)

常用属性

对齐方式：

textView.contentMode = .Topleft

内容：

textView.text = "bla bla bla ..."

字体：

textView.font = UIFont(name: ".SFUIText-Medium", size: 24)

name 属性以字符串的方式选择 iOS 字体，第二个参数为大小。

文本框：

textView.layer.borderWidth = 1

添加链接：

textView.dataDetectorTypes = UIDataDetectorTypes.None //都不加链接

属性可以替换为

UIDataDetectorTypes.PhoneNumber 只有电话加链接

UIDataDetectorTypes.Link 只有网址加链接

UIDataDetectorTypes.All 电话和网址都加

是否显示滚动条：

textView.showsVerticalScrollIndicator = true

是否可以滚动：

textView.scrollEnabled = true

当然，滚动的前提是内容的面积要大于 textView 视图的面积。

是否可以编辑：

textView.editable = true

在将 editable 设置为 true 之后，还要将将 textView 设置为第一响应者才能对其编辑。

滑动时隐藏键盘：

textView.keyboardDismissMode = .OnDrag

也可以换成其他值，监测 textView 的不同操作以隐藏键盘

防止键盘遮挡内容

如果我们的 UITextView 几乎占满了整个屏幕，那么当键盘出现的时候就会遮挡位于底部的内容。通常，解决方案有两种：

  • 当键盘出现时，在文字的底部流出足够的空行，以显示键盘
  • 当键盘出现时，压缩 textView 的高度以容纳底部的键盘

第一种方式受到文字大小的限制，而且需要额外的空间，还有可能找不到光标的位置。所以我们介绍相对来讲更为精髓的第二种方式：
