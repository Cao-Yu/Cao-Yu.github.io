数据结构学习笔记-算法的度量

时间复杂度的定义

在进行算法的分析时，算法的总执行时间 T(n) 是关于问题规模 n 的函数。记作 T(n) = O(f(n))。也就是我们常说的大 O 表示法。

在这里我们计算的时间复杂度，是对于最坏情况的考察，因为这代表着系统的鲁棒性有保证。平均情况的时间复杂度是对运算时间的一个期望，并不能表示最坏情况。

时间复杂度的计算

时间复杂度的计算，大体上遵从以下规律，实际分析当中需要具体分析。

  • 用常数 1 取代所有相加的常数
  • 只保留高阶项
  • 把最高阶项的常数置为1

平方阶

嵌套循环是典型的平方阶复杂度，我们应该由内而外的分析。内层循环执行 n 次，外层循环也执行 n 次，所以时间复杂度为 O(n^2）。

下面增加一点难度：

内层循环的执行次数随着 i 的增加而减少，所以执行次数为等差数列：

n + (n - 1) + (n - 2) + (n - 3) ... = (n * (n + 1)) / 2

化简为多项式形式：n^2 / 2 + n/2，所以按照规则，只保留最高阶项，然后再去掉常数 1/2。 所以结果仍然是 O(n^2)。

看到这里可能会感觉奇怪，为什么以上两个算法的打印语句的执行次数明显不一样，但时间复杂度却一样呢？事实上，时间复杂度的目的就是为了描述算法执行时间的数量级与 n 的关系，只是个粗略的估计。并且时间复杂度的估量是在假设 n 足够大的情况下，用极限的概念来理解的。

空间复杂度的计算

一个算法的优劣，不仅要考量时间复杂度，还要考虑对硬件存储空间的要求。

空间复杂度 S(n) = O(f(n))，其中 n 是问题的规模，f(n) 存储空间关于 n 的函数。

空间复杂度主要取决于执行算饭所需的存储空间。
