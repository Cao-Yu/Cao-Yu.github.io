数据结构学习笔记-二叉树

发表于 2016-05-13   |   分类于 数据结构学习笔记

二叉树的定义

二叉树是零个或多个节点的有限集合。

二叉树的特点

特殊的二叉树

  • 斜二叉树：所有节点只有左节点的叫左斜二叉树，反之是右斜二叉树。
  • 满二叉树：所有具有孩子的节点，都有左右两个孩子的二叉树。
  • 完全二叉树：按照层序编号后，编号与满二叉树完全一致的二叉树。

说白了，完全二叉树就是心里默默的给二叉树从左到右编号，如果没出现空位置，就是完全二叉树。完全二叉树具有向左集中的特点，如果某一个节点只有一个儿子，肯定是左儿子。

性质

  • 第 i 层最多有 2 的 i － 1 次方个节点
  • 深度为 k 的二叉树，最多有 2 的 k 次方 - 1 的节点数
  • 具有 n 个节点的二叉树的深度为 log n + 1

二叉树的存储

  • 顺序存储结构，使用数组来实现，每三个数据元素代表一个节点
  • 链式存储结构，使用链表来实现，叫做二叉链表。每个节点都有一个数据域和两个指针域。

二叉树的遍历

二叉树的遍历，基本上都要借助递归来实现

二叉树的遍历分为宽度优先和深度优先。宽度优先就是层序遍历。

二叉树的深度优先遍历大体上分为：

  • 前序遍历 : node -> left -> right
  • 中序遍历 : left -> node -> right
  • 后续遍历 : left -> right -> node
  • 层序遍历 : 每层从左到右

上边的代码就是典型的前序遍历，也就是把此节点的数据的获取放在两个儿子的前边。中序遍历和后续遍历的代码实现就是调换这三步的顺序。但是系统具体的操作是依靠栈的数据结构来实现的。

很多面试题都是已知两种功能遍历顺序来求二叉树的。规律是：必须已知二叉树的中序遍历结果，然后再已知前序或者后序遍历结果其中之一，便可以求得二叉树。

搜索二叉树(BST)

任何一个节点的左孩子都比右孩子的值小，这样的二叉树称为搜索二叉树。

堆

堆的全称叫做二叉堆(Binary Heap)。它是一种特殊的二叉树，需要满足以下两个条件：

  • 结构特性：堆必须是一颗完全二叉树。
  • 堆序特性：堆的任意节点都必须小于该节点的所有后裔。

满足以上两个条件的二叉树，称为堆(Heap)。

注意，堆不需要具有二叉查找树(BST)的左儿子必须小于右儿子的特性。且对于已知的一组数据来说，堆不是唯一的。

堆的性质

堆这种严苛的结构和顺序特性，是为了让堆具有易于查找的性质。

若任意节点的位置索引是 i：

  • 父节点的索引是 i / 2
  • 左子节点的位置索引是 2i
  • 右子节点的位置索引是 2i＋ 1

因此堆可以很容易的使用一个数组来实现。

堆的插入

堆的插入需要保持堆的两大特性。这里使用到一个上滤策略：

  • 首先在堆的下一个空位置创建一个空穴
  • 如果待插入元素的数值大于该空穴的父节点，则将空穴与父节点进行交换，以保证堆序性。
  • 迭代第二步直到位置合理
  • 将待插入元素的数据赋值给空穴

堆的删除

堆的删除通常要查询到堆的最小值并且删除，由于堆的最小值就是跟节点，所以重点就在于如何在删除跟节点后重新组合后裔。

堆的删除依然需要保持堆的两大特性。这里使用到一个下滤策略：

  • 首先在堆的跟节点创建一个空穴
  • 将该节点的较小的儿子与空穴交换
  • 迭代第二步直到空穴处于最底层
  • 将原堆中最后一个节点赋值给空穴

iOS面试题-内存管理

发表于 2016-05-07   |   分类于 iOS面试

简述

  • 管理的对象是堆区产生的对象，而不是基本数据类型。
  • 内存管理的核心概念就是引用计数器，当引用计数为 0 的时候，就会被系统回收。

什么是ARC？

ARC 是 Auto Reference Counting的缩写，即自动引用计数，由编译器在代码合适的位置中自动添加retain/Release/Autorelease/dealloc方法从而进行内存管理。

循环引用的产生原因以及解决方法

A 对象引用 B 对象作为属性，而B属性又饮用 A对象作为属性。两个对象互相持有对象，所以引用计数一直都是1。

多个对象间依然会存在循环引用问题，形成一个环，不容易察觉：

A 对象引用 B 对象，B 对象引用 C 对象，C 对象引用 D 对象，D 对象引用 A 对象。

解决办法就是在可能出现循环引用的地方，主动断开一个环节，回收对象。

或者使用弱引用。

强引用和弱引用

  • 强引用：当前对象被其他对象引用时，会执行retain操作，引用计数器+1。当retainCount=0时，该对象才会被销毁。因为我们要进行对象的内存管理，所以这是默认的引用方式。（默认是强引用）

  • 弱引用：当前对象的生命周期不被是否由其他对象引用限制，它本该什么时候销毁就什么时候被销毁。即使它的引用没断，但是当它的生存周期到了时就会被销毁。

在定义属性时，若声明为retain类型的，则就是强引用；若声明为assign类型的，则就是弱引用。后来内存管理都由ARC来完成后，若是强引用，则就声明为strong；若是弱引用，则就声明为weak。

weak严格的说应当叫“ 归零弱引用 ”，即当对象被销毁后，会自动的把它的指针置为nil，这样可以防止野指针错误。而assign销毁对象后不会把该对象的指针置nil，对象已经被销毁，但指针还在痴痴的指向它，这就成了野指针，这是比较危险的。

copy与引用的区别

  • copy 是在堆中完全复制一个对象，并且创建一个新的指针。也就是深拷贝
  • 引用就是用一个新的指针仍然指向同一个对象而已。也就是浅拷贝

简述UITableView的复用机制

  • 首先，加载占满整个屏幕高度的 cell (通常多一个或者两个)
  • 每当有 cell 滑出屏幕，便将该 cell 移到资源池(resource pool)当中
  • 有 cell 画出屏幕，就意味着有 cell 将要滑进屏幕。在 cell 将要滑进屏幕时，首先去资源池当中去寻找
  • 如果没有找到，返回nil，并且重新创建一个cell。

UIView 和 CALayer 的区别和联系

  • UIView 都是 UIResponder 的子类，所以可以响应事件。而 CALayer 直接继承 NSObject，并没有处理事件的接口。

  • UIView 是 CALayer 的CALayerDelegate。

  • 每个 UIView 内部都有一个 CALayer 在背后提供内容的绘制和显示，并且 UIView 的尺寸样式都由内部的 Layer 所提供。两者都有树状层级结构，layer 内部有 SubLayers，View 内部有 SubViews.但是 Layer 比 View 多了个AnchorPoint。

  • CALayer 是默认修改属性支持隐式动画，0.25s默认时长。在给 UIView 的 Layer 做动画的时候，View 作为 Layer 的代理，Layer 通过 actionForLayer:forKey:向 View请求相应的 action。

Move Zeroes

原题地址：Move Zeroes

问题描述：移除数组中的零，并且保持元素的顺序，且不能建立新的数组。

My Solution

遍历数组，当元素为零时，查找下一个非零元素，交换元素并且更新指针。

Better Solution

遍历数组，如果元素不为零，与 p 所指元素交换。

此法用索引 p 来标记数组的有效元素，如果有元素不为零，则赋值给有效位置。

总结

我的思路还算可以接受，但是代码过于冗杂。而且内嵌了 while 循环，增加了计算量。Better Solution 中的方法虽然每次遍历非零元素都要与自己交换一次，但是省去了判断 0 的个数和内层循环，尤其是省去了判断末尾还有没有零这个操作。

原题地址：Rotate Array

问题描述：给定一个数组和一个值，将数组中所有与该值相等的元素删去并且返回新数组的长度，可以乱序。

My Solution

不提也罢

定义两个指针，从头开始遍历数组。如果与给定元素想等，则交换并且右指针左移。如果元素不等，左指针右移。
